# Estructura del repositorio
    main   (rama principal)
        |
        |
        ▼
        develop    (rama de desarrollo)
                |  
                |  cada rama de cada integrantes
                |  se nombra con la esructura que le
                |  corresponde + su función.
                ▼
                frontend/login          (Ayelen)
                frontend/reservas       ()
                frontend/calendario     ()
                backend/usuarios        (Janina)
                backend/recepción       ()
                backend/facturación     (kevin)
                etc........
        Cada integrante trabaja en su propia rama siguiendo la estructura:
            <tipo>/<función>
            Ejemplo: backend/usuarios

# Herramientas Utilizadas
    - Xampp (con Apache PHP 8.2.12)
    - PostgreSQL
    - Composer (v2.9.5)
    - Node.js   (v24.14.0 o superior)
    ⚠️ Asegurarse de agregar PHP y Node al PATH del sistema.

# Configuración inicial para PHP
    Abrir el archivo php.ini de XAMPP (normalmente: C:\xampp\php\php.ini).
    Habilitar las extensiones necesarias eliminando ; al inicio de cada línea:
        extension=zip
        extension=pgsql
        extension=pdo_pgsql
    Guardar el archivo y reiniciar Apache.

# Clonar repositorio
    Abrir la terminal y ejecutar:
        git clone -b develop https://github.com/JaniSam/SPA_Dise-oWeb.git
    Esto clona la rama develop.
       
    Después ingresar al proyecto:
        cd SPA_Dise-oWeb

# Crear ramas de trabajo

    Para crear una rama nueva a partir de develop:

        git checkout develop
        git pull origin develop     // traer cambios de develop
        git checkout -b <nombre_rama>  // crear la rama y direccionarse a ella
    
    Cada integrante debe trabajar en su propia rama siguiendo la nomenclatura:
        <tipo>/<función>
        Ejemplo: frontend/login

    Después de hacer cambios, enviar la rama al repositorio remoto:

        git status                                      // Primero siempre revisá qué archivos modificaste
        git add .                                       // Subir todos las modificaciones 
            git add index.html                          // Subir solo el archivo index.html
            git add backend/vista/
        git commit -m "Descripción de los cambios"
        git push -u origin <nombre_rama>

# Instalar Composer y dependencias PHP
    Instalar Composer desde https://getcomposer.org
    En la carpeta backend del proyecto:
        cd spa-project/backend
        Instalar dependencias:
            composer install
    Esto descargará todas las dependencias de Laravel.

# Configuración del entorno Laravel

    Copiar archivo de entorno:
        cp .env.example .env
    Generar clave:
        php artisan key:generate

    Configurar en .env los datos de PostgreSQL:

        DB_CONNECTION=pgsql
        DB_HOST=127.0.0.1
        DB_PORT=5432
        DB_DATABASE=nombre_base
        DB_USERNAME=usuario
        DB_PASSWORD=contraseña

    Ejecutar migraciones:
        php artisan migrate

# Instalar Node y dependencias frontend
    En Windows PowerShell, permitir ejecutar scripts temporalmente:

        Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

    Ir a la carpeta frontend:

        cd C:\Libreria\SPA_Dise-oWeb\spa-project\frontend

    Instalar dependencias:
        npm install

    Ejecutar el proyecto en modo desarrollo:
        npm run dev

    Esto levantará Laravel Mix para compilar JS y CSS.

# Levantar el backend Laravel

    En carpeta backend ejecutar:
        php artisan serve
    Servidor:
        http://127.0.0.1:8000

    En la carpeta frontend:
        npm run dev

# Archivos que NO se deben subir

    Cada integrante debe crear su propio .env:
        cp .env.example .env
    
    Y configurar su propia base de datos.
    
    Gracias al .gitignore, estos archivos NO deben subirse:
        /vendor
        /node_modules
        .env
    (Cada desarrollador debe generar estos archivos localmente.)

# Treaer cambios de develop
    Asegurate de estar en tu rama
        git branch

    Traer cambios del repositorio
        git fetch origin

    Traer lo último de develop
        git checkout develop
        git pull origin develop    

    Volver a tu rama
        git checkout backend/usuarios

    Hacer merge de develop en tu rama
        git merge develop

# Integrar cambios a develop (merge)

    Cuando el trabajo en tu rama está listo:

        Ir a la rama develop:

            git checkout develop
            git pull origin develop    // traer cambios de develop 

        Hacer merge de tu rama:

            git merge backend/usuarios

        - Resolver conflictos si los hubiera
        - Commit automático si es necesario

        Hacer push de develop:

            git push origin develop
        
        ✅ Así, develop siempre tendrá la última versión de todos los cambios integrados.

        actualizar los cambios de la ramra develop
            git branch 
                deben estar posicionados en sus ramas. Ej: backend/usuarios
            git fetch origin // traer los cambios del repositorio
            git checkout develop // traer los ultimos cambios de develop
            git pull origin develop

            git checkout backend/... // volver a la rama (git branch para asegurar)
            
            git merge develop -m "merge develop into backend/usuarios"//hacer merge de develop en tu rama sin abrir el editor
